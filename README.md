# DNF NPK Protector 使用说明

## 发布方式

GitHub 仓库只作为发布页使用，不公开源码。

用户只需要从 Release 下载 `dnf_npk_protector-1.0.0-win-x64.zip`，解压后通过工具箱启动，或直接运行：

```text
bin/app/DNFNPKProtector.exe
```

Release 包内不包含源码和 README，只包含工具箱需要的 `tool.json`、主程序 exe 以及运行依赖。

## 功能边界

这个工具用于对单个补丁 NPK 做磁盘静态保护，让常见外部 NPK/IMG 工具难以直接解析真实内容，同时保持游戏读取路径不变。

它不修改游戏、不安装驱动、不需要额外运行辅助进程。游戏运行后仍然能在内存或渲染链路中看到资源，因此它不是运行时反抓取方案。

## 默认保护

默认保护会：

- 保留真实 IMG payload 的游戏可读部分。
- 对 IMG 帧表高位做保护，让游戏按低位读取仍正常，但外部工具按完整 32 位字段解析时容易失败。
- 将 NPK 目录项 declared size 写小，干扰只按目录项大小读取的工具。
- 不默认堆大量垃圾 IMG、重名 IMG 或 5000 条目，避免体积明显膨胀。
- 不默认内置恢复 manifest/key。

## 使用流程

1. 选择输入 NPK 文件。
2. 选择输出目录。
3. 按需设置 guard 名字；默认不用改。
4. 点击开始保护。
5. 用游戏实际加载保护后的 NPK 验证显示效果。
6. 再用外部工具打开确认保护效果。

## 注意

不同外部工具的解析逻辑不同，保护效果需要用目标工具实际验证。若某个工具仍能解析，需要继续针对它的读取方式增加新的兼容干扰策略。
